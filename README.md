# 灵眸(UniOps) 统一运维平台

灵眸(UniOps) 是一个基于 Spring Boot 3.2.0 开发的统一运维平台，提供便携的通用的数据表管理和HTTP调用日志记录，以及任务调度等功能。

## 项目介绍
灵眸是一款轻量级系统与接口运维平台，专为中小型系统及接口的可视化运维管理而设计。平台致力于通过简洁、统一的操作界面，高效提供任务调度控制、请求日志追踪、第三方服务调用日志分析等核心运维功能。
其核心设计围绕“统一入口、集中管理”展开：用户只需一次登录，即可进入统一的窗口化调度平台，集中管理多个项目。通过点击对应项目，即可无缝调用并操作各自独立的业务页面，彻底解决了多系统并行时页面繁杂、难以记忆的痛点。
从根本上讲，灵眸为小系统与接口的运维可视化提供了系统性解决方案，通过标准化、平台化的管理方式，显著提升了运维操作的便捷性与效率。

## 技术栈

- **后端框架**: Spring Boot 3.2.0(后续会出基于jdk8以及Springboot2.x的版本)
- **编程语言**: Java 21
- **持久层**: MyBatis-Plus 3.5.5
- **数据库**: SQL Server 2019(可无缝衔接至其他sql库)
- **连接池**: Druid
- **HTTP客户端**: OkHttp 4.12.0
- **JSON处理**: Fastjson2
- **API文档**: SpringDoc + Knife4j
- **AOP**: AspectJ

## 功能特性

### 1. 通用实体管理
- 支持动态实体的CRUD操作
- 自动识别实体主键字段
- 支持分页查询（兼容SQL Server分页语法）
- 实体缓存管理
- 自动类型转换（支持Long、Integer、Date等常见类型）

### 2. HTTP请求日志记录
- 拦截所有进入的HTTP请求
- 记录请求参数、响应消息、耗时等信息
- 记录异常栈信息
- 支持日志清理策略

### 3. 第三方HTTP调用日志
- 记录对外部服务的HTTP调用
- 包含请求地址、参数、响应等完整信息
- 支持多种HTTP方法（GET、POST、PUT、DELETE、PATCH等）
- 提供OkHttp工具类进行HTTP调用

### 4. 任务调度
- 集成Quartz任务调度框架
- 支持动态任务管理
- 提供任务执行日志
